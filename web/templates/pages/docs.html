Getting Started

Once gobot is invited to your Discord server, you must onboard it by configuring an AI provider. Without this initial setup, the /ask command will remain offline.

Note: Administrative configurations like /setkey and /config can only be executed by the verified Server Owner due to strict security profiles.

Provider Configuration

Use the /setkey command to bind your API credentials to the server. The onboarding flow requires two parameters:

  • provider: Select Gemini, OpenAI, Anthropic, or Mistral.
  • key: Paste your provider API key.

Upon submission, gobot will query the provider's endpoint to confirm the key's validity and discover the active models associated with your account. You will then be prompted to select a default model from an interactive menu.

Settings Reference

Configure bot preferences using the /config command. The primary configuration parameter is:

multi_message

  • Value: off (Default): Responses from the AI that exceed Discord's 2000-character limit are clipped to fit within a single message. The bot instructs the AI to write concise summaries to avoid overflow.
  • Value: on: Responses that exceed the 2000-character limit are automatically segmented and delivered as consecutive follow-up messages, preserving the full context.

Troubleshooting FAQ

Why am I getting "Access Denied" errors?

Certain commands are strictly scoped to the Server Owner. Standard administrators or users with elevated roles (like Administrator or Manage Server) cannot run /setkey or /config.

What happens if my API key quota runs out?

gobot will handle the quota error gracefully, logging a warning internally and returning a user-friendly error response. The bot never leaks raw API traces or credential variables during error responses.
