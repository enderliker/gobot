Cómo funciona gobot

01

BYO-API-Key

El dueño del servidor configura su API key con /setkey. Las claves se cifran con AES-256-GCM en la base de datos.

02

Pregunta e Interacción

Cualquier usuario interactúa con la IA usando /ask. La IA tiene acceso a herramientas de lectura y moderación.

03

Frontera de Seguridad

Si la IA determina que debe banear, expulsar o silenciar a alguien, gobot pausa el flujo y requiere aprobación humana.

04

Búsqueda Web en Vivo

Si requiere datos actuales, busca en la web con Tavily y te resume los resultados sin saturar el canal de links.

Construido para Servidores Exigentes

🛡️

Confirmación Humana Garantizada

La IA nunca actúa por sí sola cuando se trata de tomar decisiones administrativas en el servidor. Todas las acciones peligrosas o destructivas pasan por un riguroso paso interactivo de aprobación antes de interactuar con la API de Discord.

🔑

Cifrado en Reposo

Las API keys del servidor no se exponen ni almacenan en texto plano. Se guardan cifradas usando un wrapper de cifrado simétrico AES-GCM-256.

🔌

Multi-IA Nativo

Soporta múltiples proveedores principales: Gemini, OpenAI, Anthropic y Mistral. Cambia de modelo y proveedor al instante en caliente.

🔍

Web Search Integrado

Búsquedas inteligentes usando la API de Tavily. La IA lee y te resume las noticias o eventos deportivos más recientes al instante.

⚡

Escrito en Go y Altamente Optimizado

A diferencia de bots en Python o NodeJS, gobot es extremadamente rápido y consume pocos recursos. Dispone de un transport personalizado que previene rate limits con la API de Discord de manera asíncrona.

Autoalójalo tú mismo

gobot es de código abierto y está diseñado para ser hospedado de forma autónoma con un consumo mínimo de recursos.

La base de datos SQLite embebida en Go puro no requiere controladores CGO, permitiendo despliegues limpios en un binario estático y ligero.

Leer Guía en GitHub
docker-compose.yml 
services:
  bot:
    build: .
    container_name: gobot
    restart: unless-stopped
    env_file:
      - .env
    volumes:
      - ./data:/data

  web:
    build:
      context: .
      dockerfile: Dockerfile.web
    ports:
      - "8081:8081"
    environment:
      - WEB_PORT=8081
