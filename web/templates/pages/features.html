PRODUCT CAPABILITIES

Enterprise-Grade Control & Integration

Explore the foundational architecture and security boundaries built into gobot to ensure reliable and policy-compliant AI operations inside your Discord organization.

Human-in-the-Loop Moderation

The definitive safety feature of gobot. Destructive or administrative actions proposed by the AI—such as member bans, kicks, or timeouts—are routed to interactive confirmation dialogs. No execution occurs without a verified administrator confirming the payload.

AES-256-GCM Secure Storage

Server credentials and API keys are never stored in plaintext or exposed to unauthorized parties. The database layer utilizes standard AES-256-GCM authenticated encryption, keeping your keys private at rest.

Decoupled Multi-Provider AI Engine

Switch between the industry's leading language model providers instantly. Support for Gemini, OpenAI, Anthropic, and Mistral is built directly into the engine, allowing you to configure different providers and specific models for individual servers.

Contextual Web Search Synthesis

Equip the assistant with real-time web capabilities. Through our Tavily search integration, the bot performs live queries to retrieve current information, synthesizing the results into a natural, cohesive response without dumping raw lists of links.

Discord API Rate Limit Protection

A custom HTTP transport intercepts Cloudflare and Discord rate limit headers dynamically. Write requests are managed with minimum intervals to guarantee high performance under load without triggering service blocks.

Flexible Database Drivers

Fully compatible with SQLite, PostgreSQL, and MySQL databases. Schema migrations run automatically upon process startup to ensure database consistency across environments.

Configurable Multi-Message Outputs

Manage large responses without losing critical context. Administrators can enable multi-message mode using /config, prompting the bot to segment and deliver responses exceeding Discord's 2000-character limit across sequential follow-ups.

Security-First Prompt Sandboxing

Strict system prompt instructions and inputs sanitization shield the underlying prompt architecture from potential injection attacks and instruction leaks, maintaining the security boundary of the model.
