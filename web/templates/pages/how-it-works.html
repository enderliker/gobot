How gobot Operates

Understand the sequence of events from configuration to execution that keeps your server automated yet strictly supervised.

01

Secure Credential Onboarding

The server administrator invokes /setkey to supply their API key and configure the preferred provider. The key is validated against the provider API in real-time, encrypted with AES-256-GCM, and safely stored in the database.

02

Contextual User Queries

Members query the model using the /ask command. The query is bundled inside a prompt envelope that injects local context, system policies, and administrative permission boundaries before hitting the provider API.

03

Enforcing the Safety Boundary

If the AI response requires an administrative action (e.g. banning or muting a user), the execution halts. gobot presents an interactive verification card to the administrators. The action is held in queue and only executed when a human confirms the payload.

04

Web Query Resolution

When the query relies on real-time data, the AI invokes the integrated search tool. gobot queries the Tavily API, retrieves current web summaries, and sends them back to the AI for final natural language synthesis.
